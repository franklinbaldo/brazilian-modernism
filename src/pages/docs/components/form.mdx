# Form

The `Form` component acts as a macro-level wrapper that orchestrates multiple `FormField` instances. It supports native form actions, submission event handling, error summaries, and semantic fieldset grouping to abide by the Curva & Concreto structural requirements.

## Basic Usage

When used simply as a wrapper, it provides consistent layout gaps between form fields and standardizes the native `onsubmit` workflow.

```svelte
<script>
  import Form from 'cobogo/components/Form';
  import FormField from 'cobogo/components/FormField';
  import TextInput from 'cobogo/components/TextInput';
  import Button from 'cobogo/components/Button';

  function handleSubmit(e) {
    e.preventDefault();
    alert('Submitted!');
  }
</script>

<Form onsubmit={handleSubmit}>
  <FormField label="Email" htmlFor="email">
    <TextInput type="email" id="email" required />
  </FormField>

  <FormField label="Password" htmlFor="password">
    <TextInput type="password" id="password" required />
  </FormField>

  <Button type="submit">Login</Button>
</Form>
```

## Fieldset Grouping

By providing a `legend` prop, the form renders its content inside a semantic `<fieldset>` with a stylized `<legend>`, perfect for logically grouping related fields such as addresses or personal information.

```svelte
<Form legend="Personal Information" method="post">
  <FormField label="First Name" htmlFor="fname">
    <TextInput id="fname" />
  </FormField>
  <FormField label="Last Name" htmlFor="lname">
    <TextInput id="lname" />
  </FormField>
</Form>
```

        <TextInput id="demo-lname-2" />

    </div>

</div>

## Validation States

Form fields and their wrappers provide robust validation states adhering to the *Curva & Concreto* doctrine. Validation dynamically updates border, background, and text colors.

In **Dark Mode**, the base values (`--papel` and `--concreto`) invert, and the validation "soft" backgrounds (`--vermelho-soft`, `--verde-soft`) also invert nicely, while the strong semantic accents (`--vermelho`, `--verde`) remain intentionally vibrant to ensure clarity.

### Form-Level Errors

For situations where validation spans multiple fields or fails on the server, you can pass an `error` prop to `<Form>`. This will display a highly visible, assertive alert summary at the top of the form using the `vermelho` color scheme.

```svelte
<Form error="The username and password combination is incorrect.">
  <FormField label="Username" htmlFor="user">
    <TextInput id="user" invalid />
  </FormField>
  <FormField label="Password" htmlFor="pass">
    <TextInput type="password" id="pass" invalid />
  </FormField>
  <Button type="submit">Try Again</Button>
</Form>
```

### Field-Level Validation (Valid & Invalid)

You can pass `error` or `valid` message strings directly to the `<FormField>` wrapper. It will automatically propagate `invalid` and `valid` boolean properties down to any standard form control wrapped inside (like `TextInput`, `Select`, `Checkbox`, etc.).

```svelte
<Form>
  <FormField label="Email" htmlFor="user-email" error="This email is already in use.">
    <TextInput type="email" id="user-email" />
  </FormField>

  <FormField label="Username" htmlFor="user-name" valid="Username is available!">
    <TextInput id="user-name" />
  </FormField>
</Form>
```

## API Reference

### Props

| Prop | Type | Default | Description |
| :--- | :--- | :--- | :--- |
| `action` | `string` | `undefined` | The standard HTML form action endpoint. |
| `method` | `'get' \| 'post' \| 'dialog'` | `'get'` | The HTTP method to use for submission. |
| `legend` | `string` | `undefined` | Triggers a semantic `<fieldset>` wrap around the fields and displays this text. |
| `error` | `string` | `undefined` | Displays a prominent form-level error message at the top. |
| `onsubmit` | `(e: SubmitEvent) => void` | `undefined` | Standard submit event listener. |
| `class` | `string` | `""` | Additional custom CSS classes. |
