# Checkbox

The `Checkbox` component is the foundational atom for multiple-choice data entry in the COBOGĂ“ design system. It embodies the **Curva & Concreto** doctrine with its honest pool-shadows, sharp geometry, and unmistakably distinct validation states.

</div>

## Usage

`Checkbox` is typically wrapped by a `FormField` component for standalone validation, or within an `OptionGroup` for grouped selections.

```svelte
<script>
  import { Checkbox } from 'cobogo';
  let acceptTerms = $state(false);
</script>

<label class="flex items-center gap-2">
  <Checkbox bind:checked={acceptTerms} value="terms" />
  I accept the terms and conditions
</label>
```

## Validation States

Validation states use the standard design system validation color mapping: `--vermelho-soft`/`--vermelho` for errors, and `--verde-soft`/`--verde` for success. These states dynamically adjust based on whether the input is currently checked, providing persistent geometric feedback.

```svelte
<!-- Invalid unselected state -->
<Checkbox invalid={true} />

<!-- Invalid checked state -->
<Checkbox invalid={true} checked={true} />

<!-- Valid unselected state -->
<Checkbox valid={true} />

<!-- Valid checked state -->
<Checkbox valid={true} checked={true} />
```

## Props

| Prop | Type | Default | Description |
| :--- | :--- | :--- | :--- |
| `checked` | `boolean` | `false` | The bindable checked state of the input. |
| `value` | `string` | `undefined` | The value attribute for the input, particularly useful with `group`. |
| `group` | `any[]` | `[]` | The bindable array for managing multiple checkbox selections natively. |
| `disabled` | `boolean` | `false` | Whether the input is disabled. |
| `invalid` | `boolean` | `false` | Whether the input is in an invalid state. |
| `valid` | `boolean` | `false` | Whether the input is in a valid state. |
