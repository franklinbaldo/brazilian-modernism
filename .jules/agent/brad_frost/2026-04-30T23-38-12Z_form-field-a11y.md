## 3. THIS SESSION
- **What I did:** Refactored `FormField.svelte` to explicitly pass ARIA linkages to its child snippet. Changed the `children` prop type to `Snippet<[{ id: string; 'aria-describedby'?: string }]>`. Derived a `describedBy` state based on the presence of `error` or `helper` text. Passed these arguments down via `{@render children({ id: htmlFor, 'aria-describedby': describedBy })}`. Wrote a comprehensive BDD suite using a `FormFieldWrapper.test.svelte` component to assert these bindings.
- **Why:** The prior session correctly noted that while inputs were now structurally sound, they lacked layout context. A screen reader user encountering a `TextInput` inside a `FormField` needs the error message linked via `aria-describedby` to understand why the field is invalid. Svelte 5 snippets allow us to provide these linkages as explicit arguments rather than relying on brittle context API solutions or cloning elements.
- **Decisions made:** Chose to type the snippet arguments explicitly. Opted to create a dedicated `.test.svelte` wrapper component for testing, as rendering Svelte 5 snippets purely from TypeScript test files is cumbersome and non-idiomatic.
- **Alternatives rejected:** I rejected the idea of using the Svelte Context API to pass IDs down implicitly. Implicit context requires the child components (`TextInput`, `Select`) to actively "listen" for it, breaking their isolation as independent atoms. Passing props explicitly via snippet arguments maintains the integrity of the atoms while enforcing the molecule's structural requirements.
- **Citations to DESIGN.md:** Addressed "WCAG AA Floor", "Support keyboard navigation with explicit focus-visible states", and "Include `aria-` attributes (e.g., `aria-label`, `aria-live`, `aria-expanded`, `aria-hidden`) where appropriate".

## 4. BRASILIANA
I learned the phrase *jeitinho brasileiro*. It is the cultural knack for finding a workaround, for bending the rules just enough to make something work when the formal systems fail. Sometimes it is a charming survival mechanism; other times it is a symptom of structural dysfunction. When building UI components, we often resort to our own forms of *jeitinho*—a hacky CSS selector here, an implicit context variable there—just to get two elements to talk to each other. But a design system cannot be built on *jeitinhos*. It requires rigorous, unapologetic structure. We must build the pathways so clearly that no workarounds are necessary. The formal system must work.

## 6. NOTES TO FUTURE-ME
Testing Svelte 5 snippets in Vitest requires a wrapper component (`.test.svelte`). Don't waste time trying to mock snippet functions directly in the `render` call's props object—it leads to disconnected DOM nodes and failed role queries. Embrace the wrapper pattern; it's cleaner and mirrors real-world usage.
