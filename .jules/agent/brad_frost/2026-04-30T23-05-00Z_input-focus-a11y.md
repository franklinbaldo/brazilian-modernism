## 3. THIS SESSION
- **What I did:** Refactored \`Button.svelte\`, \`TextInput.svelte\`, and \`Select.svelte\` to accept and forward \`...rest\` attributes. For \`Button.svelte\`, I also formally typed the props to handle \`Snippet\` children and precise \`type\` values, matching Svelte 5 paradigms. Wrote extensive Vitest test suites for all three components to assert proper rendering, state-driven attributes (\`aria-invalid\`), and successful keyboard event forwarding (\`onkeydown\`).
- **Why:** The prior session set up the BDD pipeline but only tested \`Alert\`. A design system where atoms do not allow external event binding prevents the assembly of accessible molecules. The system requires keyboard navigation via \`onkeydown\`, \`Enter\`, and \`Space\` integrations. Forwarding \`...rest\` resolves this universally for these inputs.
- **Decisions made:** Chose to type \`Button.svelte\` explicitly instead of keeping loose untyped props, as \`Select\` and \`TextInput\` already utilized TS for stricter prop validation. Added BDD specifications targeting exact key presses to ensure we don't accidentally intercept or swallow keyboard navigation.
- **Alternatives rejected:** I considered manually defining \`onkeydown\`, \`onfocus\`, \`onblur\` exports for each input, but rejected this as it violates the principle of keeping components as transparent wrappers. Using the \`...rest\` spread operator in Svelte is the natively correct pattern for attribute passthrough.
- **Citations to DESIGN.md:** Addressed "WCAG AA Floor", "Support keyboard navigation with explicit focus-visible states".

## 4. BRASILIANA
The word is *gambiarra*. It has no direct English translation — "workaround" is too sterile, "hack" is too digital, "jury-rig" lacks the joy of it. It is the deeply Brazilian art of solving a problem using completely unintended materials, often out of necessity, occasionally out of sheer stubborn creativity. You see it in the wiring of a street vendor’s cart or the way a broken chair leg is replaced by a stacked pile of old tiles. Today, looking at the way modern frontend frameworks sometimes require us to jump through hoops just to pass a simple HTML attribute down to a DOM node, it felt like a digital *gambiarra*. We are overcomplicating the web, and I'm reminded that the best solutions are usually the ones that strip away the cleverness and just let the materials do what they were built to do.

## 6. NOTES TO FUTURE-ME
Svelte 5 strictly requires `import type { Snippet } from 'svelte'` for slot equivalents. The `...rest` spread approach on `Button` requires `[key: string]: any` in the TypeScript definition to avoid compiler errors regarding unknown props. Keep relying on Vitest for these interaction checks; it caught a missing spread on `TextInput` immediately during TDD.
