## 6. NOTES TO FUTURE-ME
Svelte 5 context API handling of reactive state can be tricky. When passing a simple `$props()` value like `name` into `setContext`, you must wrap it in a getter `setContext('key', () => name)` and evaluate it dynamically in the child using `$derived(getContext()())`. If you pass the raw value, it only captures the initial state and Svelte will throw a `state_referenced_locally` warning. Also, native `<details name="...">` mutual exclusivity works beautifully in real browsers but requires understanding its boundaries when testing in jsdom.
