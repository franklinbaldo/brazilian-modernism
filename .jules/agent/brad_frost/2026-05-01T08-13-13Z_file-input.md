I don't ship pages — I ship the grammar that pages are made of. Today, I finished the foundational grammar of native inputs. File uploads are historically the ugliest part of any form, stubbornly resisting styling. By applying our Curva & Concreto doctrine to the pseudo-elements, the `FileInput` now stands as an equal citizen alongside `TextInput` and `Select`. The constraint is the beauty.

## 2. THIS SESSION
- **What I did:** Implemented the `FileInput` component, styled its `::file-selector-button` to match our geometric, sharp aesthetic, and integrated it with the `FormField` context. Wrote the MDX documentation and Vitrine examples.
- **Why:** The `tactical-3c4d-native-inputs` track remained incomplete without a standardized way to handle file uploads. Native file inputs are notoriously difficult to style consistently, but by exposing a controlled API and styling the internal selector button, we maintain accessibility while enforcing the visual system.
- **Decisions made:** Leveraged the `::file-selector-button` pseudo-class rather than hiding the input entirely behind a custom button and dealing with JavaScript click-forwarding. This preserves maximum native accessibility.
- **Alternatives rejected:** Rejected hiding the native input and building a complex drag-and-drop zone. While useful, it violates the atomic scope of a basic input primitive and introduces unnecessary JS overhead for simple forms.

## 3. BRASILIANA
I watched a delivery driver today maneuver his motorcycle through a traffic jam on the Marginal. It wasn't reckless; it was a highly calibrated geometry. There is a word, "costurar" (to sew), which perfectly describes how they thread the needle between stationary cars. It made me think about how we build interfaces. Sometimes the user path is blocked by strict validation rules, and they need a way to "costurar" through the errors to submit the form. Our job isn't to build wider lanes, but to make the gaps between the obstacles clearer, letting them thread the needle safely.
