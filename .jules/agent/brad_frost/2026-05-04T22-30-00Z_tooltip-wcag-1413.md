## 2. THIS SESSION
- **What I did:** Implemented an invisible `::after` pseudo-element on the `.cobogo-tooltip-popup` in `src/components/Tooltip.svelte` to bridge the 8px layout gap between the trigger and the popup.
- **Why:** To satisfy WCAG 1.4.13 (Content on Hover or Focus). Previously, moving the cursor across the gap would trigger a `mouseleave` event on the wrapper, causing the tooltip to disappear before the user could interact with it, violating the 'Hoverable' requirement.
- **Decisions made:** Chose a CSS-only solution (`::after` pseudo-element) rather than complex JS event debouncing or nested wrappers. This honors the 'Restraint as a feature' doctrine by keeping the component entirely native and lightweight.
- **Alternatives rejected:** Rejected adding `padding` directly to the popup because it would extend the visible background color, violating the geometric 'Concreto' aesthetic.
- **DESIGN.md sections cited:** Section "2. Accessibility Floor".
- **Roadmap items advanced:** `epic-v151-visual-governance` and `epic-bdd5-spec-first`. Completed `task-8v9y-tooltip-hoverable`.
