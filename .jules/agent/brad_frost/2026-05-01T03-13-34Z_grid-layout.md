## 3. THIS SESSION
- **What I did:** Implemented the `Grid` and `Column` atom components to establish a proper 12-column global layout system, bridging modern CSS Grid with our component library. Wrote comprehensive BDD tests using Vitest and @testing-library/svelte, verifying spanning math and HTML semantics. Added an MDX documentation page under the Foundations/Components section. Fixed the CSS logic so the column start offsets do not collapse the span calculation.
- **Why:** To prevent arbitrary margins, paddings, and "magic numbers" leaking into product code, the design system must provide structural primitives. By wrapping CSS Grid into reusable Svelte atoms, we enforce consistent, responsive rhythms across the entire UI.
- **Decisions made:** Instead of an exhaustive prop API trying to reinvent every CSS Grid property, I kept it strictly to 12-columns and responsive spans (`span`, `sm`, `md`, `lg`) and `start` offsets. Both components allow overriding the underlying HTML element via the `as` prop to ensure we maintain semantic document flow and don't break WCAG AA structural guidelines.
- **Citations to DESIGN.md:** Applied the "Curva & Concreto" doctrine explicitly— grids are pure "Concreto". They establish the invisible sharp lines upon which all other elements rest.

## 4. BRASILIANA
The grid of Brasília from the sky looks like an airplane, or a bird, depending on who you ask. Lucio Costa called it the *Plano Piloto*. But on the ground, the grid is enormous, punishing, and scaled for cars, not people. Yet, in the *superquadras*—the residential blocks designed to be self-contained—there is an intensely human scale. The rigid geometry of the city plan is softened by the permeability of the blocks themselves. When designing the `Grid` and `Column` components, I tried to build *superquadras*. The 12-column structure is absolute and rigid, but the columns themselves are the pilotis. They let the content breathe underneath the math.

## 6. NOTES TO FUTURE-ME
Svelte 5 dynamic inline styles referencing CSS Custom Properties (e.g., `style="--col-span: {span}"`) is incredibly elegant for passing responsive breakpoint data down to media queries without shipping bloated utility classes. We should keep using this pattern. Be careful about `...rest` destructurings overwriting custom components props like `style` - always manually extract it and join it instead! Also, when declaring CSS shorthand `grid-column`, remember that specifying the `span X` effectively breaks down into a `grid-column-start` or `grid-column-end` implicitly, so it's safer to map to `grid-column-end` if offsets are supported.
