## 3. THIS SESSION
- **What I did:** Refactored `FormField.svelte` to use Svelte 5 `setContext` to automatically provide `id`, `aria-describedby`, `invalid` state, and `required` attributes to child inputs (`TextInput`, `Select`, `Checkbox`, `Radio`). Implemented `$derived` evaluations for the context in the child components to ensure reactivity when the wrapper's `error` prop changes dynamically. Added `.invalid` CSS styling to all inputs to apply the `--vermelho` border treatment defined in the tokens. Removed `aria-invalid` from `Radio` and `Checkbox` to comply with ARIA authoring practices and suppress Svelte compiler warnings.
- **Why:** Form validation is inherently contextual. A developer shouldn't have to manually bind the `error` state and `aria-describedby` IDs to both the wrapper and the input. By leveraging Svelte Context and strict composition, the atomic inputs simply look up their tree, realize they are inside a `FormField` that has an error, and style/configure themselves accordingly.
- **Decisions made:** Evaluated the context getter inside a `$derived` block in the child inputs. Initial instinct was to call it once on component mount, but that breaks dynamic client-side validation. Svelte 5 requires context getters to be evaluated reactively. Suppressed `aria-invalid` on checkbox/radio atoms, favoring visual `.invalid` classes, as standard screen readers derive validity from the group or fieldset level for these types.
- **Alternatives rejected:** Prop drilling. We could have added `invalid` and `describedBy` props to every input and forced the user to bind them, but that violates the "bakery" principle of implicit state coordination.
- **Citations to DESIGN.md:** Enforced the WCAG AA floor by ensuring inputs are dynamically linked to their error messages via `aria-describedby` automatically. Applied the `--vermelho` token for all danger/invalid states.

## 6. NOTES TO FUTURE-ME
Svelte 5 Context reactivity relies on passing getter functions (e.g., `setContext('key', () => value)`). Consuming components MUST evaluate this getter inside a `$derived` block (e.g., `let ctx = $derived(getContextKey()())`) to maintain reactivity; evaluating it at the top level of the script block only captures a static snapshot. Furthermore, `aria-invalid` is strictly invalid on `role="radio"` and `role="checkbox"`.
