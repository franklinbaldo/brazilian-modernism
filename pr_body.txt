**What**
Fixed WCAG AA contrast failures in the `Notification` component during Dark Mode and updated E2E testing to rigorously assert contrast ratios via relative luminance calculations.

**Why**
The design system's doctrine mandates that core identity colors (`--azul`, `--vermelho`, `--ocre`, `--verde`) remain vibrant and unchanged across color modes. However, the `Notification` component historically used these primary colors for both its text/icon and its background (`--soft` variables) under `.dark-mode`. Since the `--soft` variables invert to dark equivalents while the text variables remained very dark, this led to an invisible contrast ratio of 1.0.

**Decisions**
We decoupled readability from aesthetic branding in dark mode. While the structural accent border is preserved (using the primary identity variable), the text and icon colors are explicitly overridden inside `.dark-mode` to use `var(--fg-heading)`. This ensures WCAG AA text contrast (>= 4.5).

**Trade-offs**
The border strip in the notification has lower contrast against the background in dark mode, but the text and icon (the essential readable UI) now explicitly pass WCAG AA. We preserve the canon's constraint on color definitions while fulfilling accessibility obligations.

**Test Evidence**
- Added a robust contrast calculation to `e2e/dark-mode.spec.ts` testing for `>= 4.5` relative luminance contrast on `Notification`, `Dialog`, and `Drawer`.
- Run `npm run build`, `npx vitest run`, and `npx playwright test`. All pass successfully.
