**What**
Implemented full validation states (`valid` and `invalid`) for selection form components (`Checkbox`, `Radio`, `Switch`) and updated the `OptionGroup` structural wrapper to propagate these states downward via Svelte context.

**Why**
To finalize the Form Validation States initiative and ensure undeniable feedback (in Curva & Concreto colors) across the entire atomic input collection, strictly enforcing BDD spec coverage and visual E2E verification.

**Decisions**
- Kept the `--vermelho-soft`/`--vermelho` and `--verde-soft`/`--verde` token mapping.
- Edited `OptionGroup.svelte` to mimic `FormField`, wrapping both `error` and the newly added `valid` message in `<span>` tags accompanied by the correct semantic SVGs.
- Added visual visual assertions for the E2E Vitrine pages to avoid regressions.

**Trade-offs**
- The `valid` state takes a string or boolean, mirroring `FormField`.
- `OptionGroup` will default to `undefined-error` if no `name` attribute is provided, which is acceptable since it acts as a DOM id link.

**Test Evidence**
- Modified `features/form/validation-states.feature` to include all scenarios.
- Updated `Checkbox.test.ts`, `Radio.test.ts`, `Switch.test.ts` to assert correct class binding (`valid` / `invalid`).
- Updated `OptionGroup.test.ts` for accessibility (`closest('p')`).
- Ran Playwright visual suite in `e2e/ValidationStates.spec.ts` against the Vitrine demo: PASSED.
- Full typecheck, build, and unit tests: PASSED.
