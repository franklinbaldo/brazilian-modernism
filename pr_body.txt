What:
Refactored fluid typography tokens to use CSS `clamp()` globally.
Added BDD features and E2E specs for layout responsiveness (Grid and Column components).

Why:
To ensure typography scales fluidsly across all viewport widths natively without relying on hardcoded media queries, fulfilling the doctrine's requirement for cross-cutting fluid tokens. The added test coverage enforces the `spec-first` pipeline for layout behaviors.

Decisions:
1. Replaced static `px` values for `--t-h1` through `--t-h4` and `--t-display` with `clamp()` declarations directly in the `:root` scope of `global.css`.
2. Removed the `max-width: 768px` media query overrides for these tokens, as `clamp()` handles scaling intrinsically.
3. Added a new `features/layout/grid-fluid-typography.feature` Gherkin file.
4. Added `e2e/grid-fluid.spec.ts` which runs Playwright to verify that H1 elements actually scale down gracefully and that the `Column` component's responsive grid stacking (`grid-column-end` attributes) works as intended across desktop and mobile breakpoints.

Trade-offs:
N/A - This simplifies the CSS architecture.

Test evidence:
Ran `npx vitest run` (27 passed) and `npx playwright test` (27 passed), validating that the refactored tokens do not break layout assumptions and specifically asserting the fluid scaling of the `h1` element in the vitrine.
