**What**
Implemented the `Combobox` organism component. It combines a `TextInput` for search/filtering with a custom floating listbox. Wrote full Vitest coverage handling keyboard navigation, context inheritance, and manual selection. Created the `combobox.mdx` documentation page with a `_ComboboxDemo.svelte` Vitrine.

**Why**
The native inputs track was complete, but complex forms often require selecting from datasets too large for a standard native `<select>`. The `Combobox` fills this gap, providing an accessible, filterable dropdown organism that perfectly matches the `TextInput` aesthetics.

**Decisions made**
Avoided `$effect` for syncing the selected value back to the input during blur/close events because it caused infinite loops and race conditions in headless testing environments. Instead, I explicitly synced the state inside the `openCombobox` and `closeCombobox` functions. We used explicit `tick()` calls and mocked `scrollIntoView` to make the tests robust in `jsdom`.

**Alternatives rejected**
Rejected importing an external headless UI library (like Melt UI or floating-ui) to manage the popover. The goal of COBOGÓ is to remain dependency-free where possible, and a standard absolute-positioned listbox suffices for our current layout constraints.
