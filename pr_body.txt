**What**
Fixed a critical nested array reactivity bug in the `MultiSelect` component. Replaced the implicit `bind:group` on the inner `<Checkbox>` atoms with explicit `checked` evaluations (`value.includes(opt.value)`) and an explicit `onchange` handler to re-assign the array.

**Why**
In Svelte 5, updating an array passed through `bind:group` does not reliably trigger a re-render or properly update the bound `$bindable` variable when used inside an `#each` block that crosses component boundaries (like passing `bind:group={value}` into a child component). This caused the `$bindable` `value` array to not correctly reflect selected options when interacted with from the dropdown. Explicit state management guarantees synchronization.

**Decisions made**
- Opted for explicit native `onchange` event handling and array reassignment instead of writing complex Svelte `$effect` blocks, adhering to the Curva & Concreto aesthetic of simple, honest mechanics.

**Trade-offs**
- Slightly more verbose logic inside the Svelte template (`onchange={(e) => { ... }}`) compared to the magic of `bind:group`, but provides absolute reliability for the organism's state.

**Test evidence**
- Added a specific test case in `MultiSelect.test.ts` to ensure that clicking an already selected option in the dropdown correctly unselects it and updates the badge UI.

**Visual evidence**
