**What**
Added MDX documentation and interactive Svelte 5 demos for the `DataTable`, `FilterBar`, and `Timeline` components in the vitrine. Updated BDD feature specs and Playwright E2E tests to verify their presence.

**Why**
To close the final major documentation gap in `epic-c0b0-vitrine`. Without these pages, consumers cannot see how complex structural components map to the "Curva & Concreto" geometric doctrine.

**Decisions & Trade-offs**
- Used `client:load` for the interactive demos to ensure Svelte 5 `$state` reactivity works correctly within the Astro MDX environment.
- Documented `DataTable` as an expression of the rigid "Concreto" aesthetic, while `FilterBar` uses the "Curva" pill geometry.
- No new dependencies were introduced. The demos compose existing atoms and Svelte primitives.

**Test Evidence**
- ✅ `npx vitest run` passed (147 tests)
- ✅ `npx playwright test` passed (asserts all 8 remaining component docs render correctly at their routes)
- ✅ `npm run build` & `npx astro check` completed successfully

**Visual Evidence**
- Interactive demonstrations are visible on `/cobogo/docs/components/data-table`, `/cobogo/docs/components/filter-bar`, and `/cobogo/docs/components/timeline`.
- Component styles adhere strictly to existing tokenized layouts (no new CSS classes were introduced outside of the demo containers).
