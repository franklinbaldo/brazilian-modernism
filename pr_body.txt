**What**
Implemented the `Form` component to wrap multiple `FormField`s. Added native `onsubmit` proxying, macro-level error summaries mapped to the `vermelho` admonition style, and semantic `fieldset`/`legend` rendering. Wrote full Vitest test coverage and documentation (`form.mdx`). Expanded test coverage across multiple component tests and fixed a typo in `docs/components/form.mdx` layout string to resolve build issues.

**Why**
To fulfill the broader `epic-1a2b-form-ergonomics` goal, we needed a macro-wrapper that enforces our spacing conventions (1.5rem gap) and handles structural concerns (fieldset grouping) that are tedious to implement manually every time. This guarantees that forms built with COBOGÓ automatically inherit our geometric restraint.

**Decisions made**
Avoided intercepting and replacing standard submit event properties (like `action` or `method`) entirely, preferring native HTML attributes but exposing an optional wrapper for client-side frameworks via the `onsubmit` Svelte handler. Designed the `error` state to assertively display using an `aria-live="assertive"` alert region to ensure screen readers immediately announce form-level failures.

**Alternatives rejected**
Rejected adding complex internal form state management (like tracking which specific field failed inside the `Form`). The `Form` remains visually and semantically responsible for the *macro* state, leaving micro-state to individual `FormField`s, adhering to the "atoms first" doctrine.
