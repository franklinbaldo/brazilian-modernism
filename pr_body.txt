**What**
Implemented `valid` states and `--verde` aesthetic tokens across all advanced form components: `MultiSelect`, `Combobox`, `DatePicker`, `FileInput`, and `TimePicker`.

**Why**
To fulfill the Form Validation States behavioral requirements (BDD) and ensure visual accountability across the entire form control ecosystem, establishing a single source of truth for validation context parsing and styling priority.

**Decisions**
- Extended `MultiSelect`, `Combobox`, `DatePicker`, `FileInput`, and `TimePicker` to accept `valid` props and read `valid` boolean signals from the `cobogo-form-field` context.
- Prioritized `invalid` visual states over `valid` states uniformly.
- Styled `::file-selector-button` for `FileInput` specifically to reflect the `--verde` success state on the button border.

**Trade-offs**
- To maintain API symmetry with the `FormField`, individual components parse the validation logic directly and apply specific `.valid` classes rather than having the `FormField` inject CSS dynamically. This retains Svelte style scoping.

**Test Evidence**
- Added Unit tests checking context inheritance, prop reactivity, and invalid precedence in Vitest for all modified components.
- Added E2E spec `e2e/advanced-validation-states.spec.ts` capturing visual states in Playwright. All tests passing locally (`npx vitest run`, `npx playwright test`).
